# [極簡博客維護小技巧](https://github.com/myogg/Gitblog/issues/90)

極簡博客維護小技巧
1. 內容更新

README.md 為核心：你現在的頁面直接讀取 README.md，所以所有文章、筆記、日記都可以直接寫在這裡。

保持標題結構：

# → 首頁大標題（自動作為 <title>）

## → 章節標題

### → 子標題

建議：每篇文章保持一個段落，短小精悍，避免過長的連續文字，符合極簡風格。

2. 圖片和媒體

小心圖片大小：最好用壓縮過的圖片，避免頁面加載慢。

建議使用 Markdown 語法：

![描述](images/example.png)


圖片存放於倉庫內 images/ 文件夾，保持相對路徑。

3. 維護版權信息

你現在的 deploy.yml 已自動生成年份版權：

© 2025 MyOGG. All rights reserved.


不需要手動修改年份，每次部署自動更新。

4. 部署與自動更新

自動部署：每次修改 README.md 都會觸發 GitHub Pages 更新。

手動部署：可在 GitHub → Actions → Deploy GitHub Pages → Run workflow

Tip：如果偶爾發現更新沒生效，可手動運行 workflow 確保最新內容生效。

5. 頁面樣式微調

保持極簡：CSS 只控制：

最大寬度 .markdown-body

內邊距、背景、圓角

夜間模式顏色自動切換

手機響應式

小亮點：

彩色細線條作為頁眉線條（可選）

不增加 Logo 或繁瑣動畫

6. SEO 與分享

<title>：自動取 README 第一個 # 標題，可用來作為網站標題。

<meta name="viewport">：手機自適應

如果將來想分享文章，可使用每篇文章的 GitHub Issue URL，保持極簡，不必生成單獨頁面。

7. 備份

你的 workflow 會生成備份 BACKUP/ 文件夾（每個 Issue 對應一個 Markdown 文件）。

建議：

定期 push 到另一個私有倉庫作二次備份

保留重要歷史文章，方便回溯

8. 未來小升級

保持極簡原則：

彩色線條、細微背景變化都可以，但避免複雜動畫和文章列表

小交互（可選）：

文章內鏈接高亮

簡單懸停效果

滑動回頂按鈕

🔹 總結

極簡 GitHub Pages 博客核心是：

內容為王：README.md 就是內容

極簡樣式：夜間模式 + 手機自適應 + 小彩色線條

自動化：修改即部署，省時省力

美觀但不花哨：符合你的初心
