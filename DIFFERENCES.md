# 新旧版本差异说明

您好！这是为您定制的 React 版博客系统。我们尽可能保留了您原有的设计风格（極簡、黑白配色、Work Sans 字体、时间轴布局），同时在底层技术上做了彻底的现代化改造。

以下是主要的差异对比：

## 1. 核心架构

| 特性 | 原版 (Python 静态生成) | 新版 (React 单页应用) |
| :--- | :--- | :--- |
| **数据源** | GitHub Issues | GitHub Issues |
| **生成方式** | 本地运行 Python 脚本生成 HTML | **实时拉取**：访客打开网页时直接从 GitHub API 获取数据 |
| **更新流程** | 写 Issue -> 运行脚本 -> 推送 HTML | **写 Issue -> 刷新网页即见** (无需任何构建操作) |
| **部署依赖** | 需要 Python 环境 | 只需要静态文件托管 (GitHub Pages/Vercel/Netlify) |

## 2. 功能改进

- **评论系统**：保留了您熟悉的 **Waline** 评论系统，无缝迁移。
- **搜索功能**：
  - 原版：依赖生成的 `search.json` 索引，内容更新后需要重新生成索引。
  - 新版：直接调用 **GitHub Search API**，实时搜索最新的 Issue 内容，无需维护索引文件。
- **移动端适配**：
  - 新版使用了响应式设计框架，在手机访问时会自动折叠导航菜单，布局更适合触摸操作。
- **暗色模式**：
  - 新版增加了明暗主题切换（右上角按钮），自动适配系统设置。

## 3. 设计还原

我们仔细阅读了您的 `style.css`，并在新版中进行了以下复刻：
- **字体**：引入了 `Work Sans` 和 `JetBrains Mono` (用于日期)。
- **布局**：首页保持了经典的 [年份] -> [日期 文章标题] 的列表布局。
- **宽度**：内容区域限制在 700px，保持最佳阅读体验。
- **关于页**：完全保留了您原有的排版、头像和友情链接样式。

## 4. 如何使用

1. **写文章**：在 GitHub 仓库新建 Issue 即可。
2. **修改配置**：在 `src/config.ts` 中修改博客标题等信息。
3. **部署**：将代码推送到 GitHub，GitHub Actions 会自动构建并发布。

希望这个新版本能为您带来更流畅的写作和维护体验！
